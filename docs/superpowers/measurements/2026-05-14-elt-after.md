## Headline result

**`/api/stress-test` warm dropped from ≈2.0s to ≈0.9-1.0s** — about 50% reduction.

The plan's aspirational "<50ms" was optimistic; warm latency is now dominated by:
- Portfolio holdings query (DB)
- Per-asset `get_current_price` DB reads (now batched to <10 ms each)
- Pandas computation in `run_simulation` (normalize, weight, MDD calc)

The yfinance fetch — previously the dominant cost — is now eliminated on warm cache. The remaining ~1s is honest server-side compute.

### Cold path (first request after Render free-tier restart)

5.37s — both COVID and BEAR scenarios hit cache miss in sequence:
- `SCENARIO COVID → yf.download(...)` ≈2-3s
- `SCENARIO BEAR → yf.download(...)` ≈2-3s
- Then results cached.

UptimeRobot pings `/api/healthz` only, not stress-test, so cache goes cold whenever Render free-tier swaps the process. **Daily cron warmup** (TODO comment at top of `stress_service.py`) would eliminate this — wire `_fetch_scenario_closes` calls into the daily cron pipeline so cache is always warm before real-user traffic.
