Latency budget · v2.4 cache strategy

Date-keyed SystemCache (no TTL). T1 raw series + T2 computed snapshot. T3 envelope composes fresh per request (~35ms). T4 RSC unstable_cache.

Cold path
T1 miss
2–5s
FRED + Yahoo HTTP fetch (13 series — Net Liquidity, M2, NFCI, Real Yield, T10Y2Y, T10Y3M, CPI, Core PCE, GDP, NFP, Sahm Rule, VXN, Credit Spread) → state classification → bucket aggregation → score explain → DB queries → envelope. Once per date; cron warm-up at 21:00 UTC means user calls almost never hit cold path.
Warm path
T1 hit, T2 miss
~50ms
Skip external fetch. Recompute snapshot from cached series.
Hot path
T2 hit
~35ms
Default user experience after the daily cron run.
Cache flow recap
External APIs → T1 raw series (date-keyed) → T2 computed snapshot (date+meta version-keyed) → composition (no cache, ~30ms) → envelope → Next.js RSC unstable_cache (24h revalidate, tag-invalidatable on freeze + transactions)
